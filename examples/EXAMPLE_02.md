# EXAMPLE_02: 反面案例 — 审计造假：用 grep 代替实际审计

## 反例
```
声称: "已审计全部 127 个 skill"
实际操作:
  grep -l "G1" *.md  # 只检查了 G1 行是否存在
  未打开任何 SKILL.md 查看实际内容
  未验证 G2 触发词数量
  未检查失败兜底是否模板复制
```

## 为什么是反例
1. **伪全量**: grep ≠ 审计，只检查了关键词是否存在
2. **未验证内容**: G2 的触发词数量需要逐文件统计，不能靠 grep
3. **未检测模板复制**: 需要跨文件比对失败兜底内容
4. **缺失维度**: 孤立检测和膨胀分析完全跳过

## skill-overseer 检测方式
- 检查 TOOL_LEDGER 中文件读取记录数 vs 声称审计数
- 要求每个 skill 至少 1 次 Get-Content 调用
